Forge

Team Payments -- 165 min -- Thu Apr 30, 13:00
One team. One idea. Under 3 hours. Ship or scrap.
The whole team picks one idea and builds it together. Not competing teams, not a spec review. One thing, built by everyone, with a live demo at the end. Claude Code accelerates the build. Humans make the decisions.

Pitch + Pick
13:00 - 13:15 (15 min)
Anyone pitches an idea in 60 seconds. No prep, no slides. Can be anything: an internal tool, a game, a prototype, something absurd. Only constraint: it needs frontend, backend, and design work.
Voting: Dot-vote (2 dots each), pick the winner. If it's close, Ismayil breaks the tie. Move fast - don't deliberate.
Shape
13:15 - 13:35 (20 min)
Whole team at the whiteboard. PMs define what "done" looks like, scoped to 90 min of build time. Luke sketches the UI. Engineers call out what's feasible and how to split the work.
Output: A shared sketch on the whiteboard and a task list on sticky notes. Each sticky = one chunk of work someone can grab. Assign nothing - people self-select during build.
Build
13:35 - 15:05 (90 min)
Everyone works. Grab a sticky, build it, grab the next one. One shared room, music on. No stand-ups, no check-ins. If you're blocked, shout. Claude Code is the accelerator - engineers use it to move fast. Decisions stay human.
Ismayil: Float between groups, unblock, keep energy up, watch the clock. Call "30 min left" and "10 min left" loudly.
Demo + Destroy
15:05 - 15:30 (25 min)
PMs present what was built. Live demo - show it working (or failing spectacularly, which is equally fun). Then the team decides: keep it or ceremonially destroy it. Either way, the point was building it together.
Format: Vish or Nicole narrates the demo (5 min). Team reacts (5 min). Each person shares one highlight from the build - a moment, a hack, a breakthrough (10 min). Vote: keep or destroy (5 min).

Roles During Build

Vish + Nicole (PM)
Define "done." Write copy and content. Test flows as real users. Manage scope - cut ruthlessly when time runs short. Prepare and deliver the demo.
Luke (Design)
UI/UX decisions. Sketch during Shape, produce assets during Build. Visual polish. Pair with FE on layout and interaction details.
Tural + Hanna (FE)

Valerii + Habil + Gabil + Marat (BE)
Build the backend, API, data layer. Use Claude Code to accelerate. Agree on API shape with FE early in build, then work in parallel. Everyone tests their own work.
Dimitris (Platform)
Get it running. Set up the environment, deployment, infra. If there's time, add basic observability (his domain). The one who makes the demo possible.

Ground Rules

One team, one thing. No splitting into competing groups. Everyone contributes to the same build.

Scope is sacred. PMs have final say on what gets cut. If something won't fit in 90 min, it's out.

Claude Code is a tool, not a teammate. Use it to generate boilerplate, scaffold, debug. Don't use it to make product decisions.

It doesn't need to be perfect. A working demo with rough edges beats a polished spec with no output.

It doesn't need to survive. The point is the experience of building together, not the artifact.

Moderator Notes

  • Pitch + Pick must stay at 15 min. If you let it run long, you eat into build time and energy drops. Fast decision, move on.
  • During Shape, actively assign Luke to sketch on the whiteboard while PMs talk scope. Parallel work starts here, not after.
  • The first 15 min of Build is chaos - that's normal. People figure out their lane. Resist the urge to over-coordinate.
  • At the 45-min mark of Build (14:20), check progress. If the scope is too big, pull PMs aside and cut. Protect the demo.
  • Music matters. Put on a playlist. It signals "this is not a meeting."
  • Demo must be live, not a walkthrough of code. If the thing crashes during demo, that's a story too.
  • The "keep or destroy" vote at the end is the release valve. It makes the whole thing feel low-stakes and fun.
  • Photograph the whiteboard from Shape. It's a great artifact regardless of what happens to the code.
